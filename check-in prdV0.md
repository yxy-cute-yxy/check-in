# 产品需求文档 (PRD Lite)
## 项目名称：智慧工地人员实名制登记与考勤打卡系统 (演示 Demo)

---

## 1. 项目概述与技术架构
本项目是一个具备多端自适应、响应式的单页应用（SPA）原型。核心目的在于向考核官演示“新工人实名制引导”、“现场拍照考勤打卡闭环”以及“管理员/工人双视角数据实时联动”的功能，同时沉淀 AI 协作开发记录。

### 核心技术栈
- **前端框架**：Vite + React (TypeScript) —— 利用 React 丰富的生态组件提升 Vibe Coding 的开发效率。
- **UI 组件库**：Tailwind CSS（用于快速调整样式） + **Shadcn UI** 或 **Ant Design** —— 提供自带的表单、状态指示、通知以及高级打卡界面组件，确保视觉专业感。
- **数据流与存储**：纯前端 `LocalStorage` 实现模拟本地数据库。所有数据的增删改查操作在页面刷新后不丢失，且必须即时、响应式地同步更新页面状态。
- **在线部署**：**Vercel** 或 **Netlify** —— 完美支持 Vite 编译出的纯静态产物（`dist`），一键导入 GitHub/Gitee 仓库，3 分钟即可生成全局可访问、免域名、免备案的免费体验链接。

### 自适应表现
采用**移动端优先（Mobile-First）**的设计。在手机端自动 100% 满屏显示；在电脑大屏端，内容区域自动在屏幕中央居中，最大宽度限制在 `max-w-xl`（约 480px），并带有优雅的背景阴影与卡片质感，确保两端前端画面自适应且显示效果高度一致。

## 2. 视觉设计与 UI 风格指南 (供 AI 语义化理解)
请严格执行以下基于 Tailwind CSS 的语义化色彩与样式规范，打造现代化、高触感的工业级管理美感：

- **全局底色**：纯白卡片背景 (`bg-white`)，搭配极淡、清爽的灰绿色背景 (`bg-slate-50` 或 `bg-zinc-50`)，营造智慧工地的工作空间感。
- **主色调 / 安全状态（成功、签到、正常）**：科技感极强的亮绿色/荧光绿 (`text-lime-500`, `bg-lime-500` 或 `emerald-500`)。用于核心打卡按钮、出勤率环形图、以及“正常出勤”标签。
- **警示色 / 异常状态（迟到、早退、异常）**：极其醒目的安全暖橙色/琥珀色 (`text-orange-500`, `bg-orange-500` 或 `amber-500`)。**此颜色仅用于标注考勤异常**，使其在黑白绿的主色调中瞬间跳出。
- **字形与卡片**：使用深炭黑 (`text-neutral-900`) 作为标题。所有组件卡片必须使用大圆角 (`rounded-2xl` 或 `rounded-3xl`) 结合微弱的高级阴影 (`shadow-sm` 或 `shadow-md`)。

## 3. 核心业务流程与角色管理

### 首次进入拦截（新手引导闭环）
- **控制逻辑**：应用挂载（Mount）时，自动检查 `localStorage.getItem('currentUser')`。
- **拦截行为**：如果当前无任何登录工人信息（为空），系统自动拦截主页，直接且唯一渲染 **【新工人实名制登记】** 界面，强制考核官体验首个核心模块。
- **放行条件**：登记成功后，数据写入 LocalStorage 并设为 `currentUser`，页面无缝响应式切换跳转至“智慧工地工作台”主页。

### 双角色切换系统
- **位置**：工作台首页的顶部 Header 区域。
- **UI 元素**：一个设计精致的分段切换器（Segmented Toggle 按钮，高亮切换：工人模式 / 项目经理模式）。
- **联动效果**：切换身份时，即时改变全局 `currentRole` 状态，下方的工作台卡片流瞬间完成动态替换。

## 4. 功能模块详细规格

### 模块 0：智慧工地工作台 (首页入口)
- **卡片式首页布局**：点击卡片平滑进入子页面，子页面顶部自带“返回工作台”按钮。
  - **【工人模式】展示三张卡片**：
    1. 【注册信息】（查看/修改个人实名制资料）
    2. 【考勤打卡】（每日上下班打卡主战场）
    3. 【考勤记录】（个人多维度日历与统计数据）
  - **【项目经理模式】展示三张卡片**：
    1. 【注册信息】（查看全工地所有工人实名制名册）
    2. 【考勤打卡】（供管理员模拟日常工人打卡测试）
    3. 【考勤数据统计】（宏观可视化简报 + 全员考勤大总表数据联动）

---

### 模块 1：工人实名制登记
- **表单字段**：姓名、手机号（11位正则校验）、身份证号（18位标准正则校验）、工种（下拉菜单，包含：木工、电工、水泥工、焊工等）、所属班组、头像照片（设计一个模拟拍照/本地上传的点击区域，上传后页面立即展示正方形圆角预览图）。
- **数据处理**：追加保存至 `localStorage.getItem('workers')`，同时将该用户设为 `currentUser`。弹出亮绿色成功提示（Toast），返回主页。

---

### 模块 2：考勤打卡界面 (含跳转拍照验证)
- **基本视觉**：上方显示实时更新到秒的动态时钟（HH:MM:SS）。中间是一个大圆形亮绿色打卡按钮。按钮上方提供分段选择器供工人自主手动选择 **[ 上班打卡 ]** 或 **[ 下班打卡 ]**。
- **跳转拍照核心流程（升级）**：
  1. 当工人点击中间的“圆形打卡按钮”时，系统**不直接完成打卡**，而是平滑跳转/弹出一个**【现场拍照验证】**界面。
  2. 该界面调用浏览器的摄像头 API (`navigator.mediaDevices.getUserMedia`) 实时展示摄像头画面。
  3. 界面正下方提供一个“相机快门”图标，点击即触发 `canvas` 截取当前帧画面。
  4. 拍摄完成后，展示照片预览，并转为绿色标签 `"定位成功：已进入项目南门施工区"`（闪烁提示 `"正在获取双频 GPS 定位..."` 后切换）。
  5. 工人点击预览下方的“确认打卡并提交”按钮，系统正式执行打卡逻辑，将打卡时间、打卡类型、GPS位置以及**拍摄的照片（Base64 字符串格式）**一并写入 `localStorage.getItem('attendance_logs')`。
- **连续打卡小圆点连线**：在打卡主页下方，设计一条水平的连线时间轴，轴上有 5 个相连的小圆点节点。打卡成功的日子显示为亮绿色小勾或圆点，未打卡显示置灰。

---

### 模块 3：工人的考勤记录日历与统计
- **自制月历视图**：针对当前月份渲染一个精美网格日历。每个日期格子里显示日期数字，数字下方根据当天的考勤数据动态渲染**语义化彩色小圆点**（绿色代表当天考勤完全正常；橙色代表考勤异常：早上 08:00 之后打卡判定为【迟到】，下午 18:00 之前打卡判定为【早退】）。
- **个人数据微报卡片**：紧跟在日历下方，用高对比度小方块展示三组指标：[ 成功打卡 X 天 ]、[ 请假 X 天 ]、[ 迟到早退 X 天 ]。
- **周期切换器**：提供胶囊状的切换按钮 **[ 每周 ] / [ 每月 ] / [ 每季度 ]**，点击可自由切换下方统计数据的计算维度。

---

### 模块 4：项目经理考勤数据统计 (数据大闭环)
- **实时联动机制**：此页面必须与 LocalStorage 实时绑定。如果考核官在工人模式刚完成拍照打卡，切到管理员模式时，数据必须**瞬间刷新**。
- **上方：可视化宏观简报**：用纯 Tailwind CSS 手绘三个数字看板块：【今日总到岗率（百分比环形图）】、【已登记劳务总人数】、【今日考勤异常人数（亮橙色高亮）】。
- **下方：全员考勤大总表**：编写一个表格，列出全工地所有已注册工人的姓名、工种班组、今日上班打卡时间、今日下班打卡时间、以及【今日状态】（胶囊标签：正常显示为浅绿底深绿字，迟到/早退显示为浅橙底深橙字）。
